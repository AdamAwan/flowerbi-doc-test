# Integrating FlowerBI into a React Application

This guide covers the end-to-end process of connecting a FlowerBI query backend to a React frontend using the `flowerbi-react` package. You will learn how to install the package, configure the data-fetching layer, use the `useFlowerBI` and `usePageFilters` hooks, and bind query results to custom components.

## Prerequisites

- A React application (created with Create React App or any modern setup).
- A running FlowerBI-compatible API endpoint (e.g., a .NET Core server exposing a single POST route that accepts a `QueryJson` payload and returns a `QueryResultJson`). See the [server documentation](https://earwicker.com/flowerbi/) for setup details.
- TypeScript is recommended but not required; all examples use TypeScript for type safety.

## Installation

Install the `flowerbi-react` package from npm. It has a peer dependency on `react` (versions 16.13.1, 17.0.2, 18.0.0, or 19.0.0).

```bash
npm install flowerbi-react
```

If you plan to use the optional `flowerbi-react-chartjs` package for Chart.js integration, install that as well:

```bash
npm install flowerbi-react-chartjs
```

## Setting Up the Fetch Function

FlowerBI's core client library (`flowerbi`) defines a `QueryFetch` type: a function that accepts a `QueryJson` object and returns a `Promise<QueryResultJson>`. You must implement this function to POST the query JSON to your API endpoint. Below is a minimal example using the native `fetch` API without authentication headers.

Create a file `src/flowerbiFetch.ts`:

```ts
import { QueryJson, QueryResultJson } from 'flowerbi';
import jsonDateParser from 'json-date-parser';

const API_URL = 'https://your-api.example.com/query'; // replace with your endpoint

export async function flowerbiFetch(queryJson: QueryJson): Promise<QueryResultJson> {
  const response = await fetch(API_URL, {
    method: 'POST',
    cache: 'no-cache',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(queryJson),
  });

  if (!response.ok) {
    // Handle error appropriately – returning an empty result is safe
    return [];
  }

  const text = await response.text();
  // Use jsonDateParser to correctly deserialize dates
  return JSON.parse(text, jsonDateParser);
}
```

> **Rationale**: The fetch function is the single integration point between the client and server. The example is adapted from the [package documentation](https://earwicker.com/flowerbi/typedoc/flowerbi) and source repository's `home.md` file.

## Using the `useFlowerBI` Hook

The `useFlowerBI` hook manages the lifecycle of a single query: it sends the query to the server when the component mounts or when the query definition changes, and exposes the resulting data, loading state, and errors.

### Hook Signature

```ts
function useFlowerBI<TColumns extends Record<never, never>>(
  fetch: QueryFetch,
  query: IQueryDefinition<TColumns>
): {
  loading: boolean;
  error: Error | undefined;
  records: IQueryResult<TColumns>['records'];
  execCount: number;
  queryJson: QueryJson;
}
```

- `fetch`: the fetch function you defined earlier.
- `query`: a query object built using the generated TypeScript schema (or manually). It contains `select`, `filters`, `aggregations`, etc.
- Returns:
  - `loading`: `true` while the query is in flight.
  - `error`: any error thrown during fetch.
  - `records`: an array of strongly-typed result objects once loaded.
  - `execCount`: number of times the query has been executed.
  - `queryJson`: the serialised JSON sent to the server (useful for debugging).

### Example: Simple Customer Bug Count

Assume you have a schema with `Customer` and `Bug` tables, and generated TypeScript constants like `Customer.CustomerName` and `Bug.Id`. The following component fetches the number of bugs per customer and renders a list.

```tsx
import { useFlowerBI } from 'flowerbi-react';
import { flowerbiFetch } from './flowerbiFetch';

// These constants are typically generated by FlowerBI.Tools
import { Customer, Bug } from './generatedSchema';

function CustomerBugList() {
  const { records, loading, error } = useFlowerBI(flowerbiFetch, {
    select: {
      customer: Customer.CustomerName,
      bugCount: Bug.Id.count(),
    },
    filters: [],
  });

  if (loading) return <p>Loading...</p>;
  if (error) return <p>Error: {error.message}</p>;

  return (
    <ul>
      {records.map((r) => (
        <li key={r.customer}>
          {r.customer}: {r.bugCount} bugs
        </li>
      ))}
    </ul>
  );
}
```

### Example: Using Filters

You can add filters to the query object. Filters are arrays of `IFilter` objects, typically created using column comparison methods like `.equalTo(value)`.

```tsx
const { records } = useFlowerBI(flowerbiFetch, {
  select: {
    customer: Customer.CustomerName,
    bugCount: Bug.Id.count(),
  },
  filters: [Workflow.Resolved.equalTo(true)],
});
```

> **Note**: If you have multiple aggregations with different filters, see the [multi-aggregation documentation](https://earwicker.com/flowerbi/docs/full-joins.html) for the `fullJoins` flag.

## Using the `usePageFilters` Hook

The `usePageFilters` hook allows multiple chart components on the same page to interact through a shared set of filters. This enables filtering across components without prop drilling.

### Hook Signature

```ts
function usePageFilters(
  initialFilters?: ITableFilter[]
): {
  filters: ITableFilter[];
  addFilter: (filter: ITableFilter) => void;
  removeFilter: (filter: ITableFilter) => void;
  clearFilters: () => void;
}
```

- `initialFilters`: optional initial set of filters.
- Returns methods to add, remove, or clear filters, and the current list.

### Example: Shared Date Range Filter

Suppose you have two charts on a dashboard that should both respond to a date range picker.

```tsx
import { usePageFilters } from 'flowerbi-react';
import { DateRequested } from './generatedSchema';

function DateRangePicker() {
  const { addFilter, removeFilter, filters } = usePageFilters();

  const handleDateChange = (start: Date, end: Date) => {
    // Remove any previous date range filters
    filters
      .filter((f) => f.column.table === 'DateRequested')
      .forEach((f) => removeFilter(f));

    // Add new range filter (example assumes .between method exists, adjust to your API)
    addFilter(DateRequested.Id.between(start, end));
  };

  return <input type="daterange" onChange={(e) => handleDateChange(...)} />;
}

function Chart1() {
  const { filters } = usePageFilters();
  const { records } = useFlowerBI(flowerbiFetch, {
    select: { /* ... */ },
    filters, // use the shared filters
  });
  // render chart
}

function Chart2() {
  const { filters } = usePageFilters();
  const { records } = useFlowerBI(flowerbiFetch, {
    select: { /* ... */ },
    filters,
  });
  // render another chart
}
```

> **Note**: The `usePageFilters` hook is part of the `flowerbi-react` package and is described in its [README](https://github.com/danielearwicker/flowerbi/tree/master/client/packages/flowerbi-react).

## Binding Results to Custom Components

You are not limited to the built-in `FlowerBITable`. Since `records` is a plain array of strongly-typed objects, you can map it to any component – a list, a Chart.js chart, a bar chart from a library of your choice, or a Material-UI table.

### Example: Chart with Chart.js

Using the `react-chartjs-2` wrapper for Chart.js, you can pass the records directly to chart data.

```tsx
import { Bar } from 'react-chartjs-2';

function CustomerBugChart() {
  const { records } = useFlowerBI(flowerbiFetch, {
    select: { customer: Customer.CustomerName, bugCount: Bug.Id.count() },
    filters: [],
  });

  const chartData = {
    labels: records.map((r) => r.customer),
    datasets: [
      {
        label: 'Bugs',
        data: records.map((r) => r.bugCount),
      },
    ],
  };

  return <Bar data={chartData} />;
}
```

If you use the `flowerbi-react-chartjs` package, it provides pre-built components that handle mapping automatically. See its [API documentation](https://earwicker.com/flowerbi/typedoc/flowerbi-react-chartjs).

### Example: Custom Table

```tsx
function CustomerTable() {
  const { records } = useFlowerBI(flowerbiFetch, {
    select: { customer: Customer.CustomerName, bugCount: Bug.Id.count() },
    filters: [],
  });

  return (
    <table>
      <thead>
        <tr><th>Customer</th><th>Bugs</th></tr>
      </thead>
      <tbody>
        {records.map((r) => (
          <tr key={r.customer}>
            <td>{r.customer}</td>
            <td>{r.bugCount}</td>
          </tr>
        ))}
      </tbody>
    </table>
  );
}
```

## Step-by-Step Integration Summary

1. **Install** `flowerbi-react` via npm.
2. **Create a fetch function** that POSTs `QueryJson` to your server endpoint.
3. **Define queries** using generated schema constants or manual column references.
4. **Use `useFlowerBI`** in a component to execute the query and receive `records`.
5. **Use `usePageFilters`** when multiple components need to share filters.
6. **Render** the `records` array using any React component (tables, charts, lists).
