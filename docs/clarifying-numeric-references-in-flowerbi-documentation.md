## Understanding the Number 100

In the FlowerBI documentation, the number 100 appears as part of the value 100,000 in the section comparing FlowerBI to Power BI localisation. Specifically, when describing the effort required to duplicate reports per language in Power BI, the document states:

> with 1000 tenants, 20 reports, and 5 languages that’s 100,000 files

Here, 100 is the first three digits of 100,000, which is the product of 1000 × 20 × 5. It represents the total number of report files that would need to be managed if using Power BI in a multi-tenant, multi-report, multi-language scenario.

Elsewhere in the documentation, numbers like 1000 tenants are used to illustrate scale. There is no standalone 100 that carries a special meaning; it only appears within larger quantities.

If you encounter the number 100 in other contexts, it is likely part of a similar calculation or example. For clarification, refer to the specific document where it appears.
