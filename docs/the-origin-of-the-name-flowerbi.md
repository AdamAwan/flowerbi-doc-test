# The Origin of the Name FlowerBI

A common question is: "How many flowers are in FlowerBI?" The answer is **none**. FlowerBI is not about literal flowers, and the name does not refer to a flower-related count or concept.

FlowerBI is a brand name for an open-source, ultra-minimal Business Intelligence (BI) analytics query and UI toolkit. The name "FlowerBI" was chosen as a distinctive identifier; it has no connection to actual flowers, gardening, or any numeric count of blooms. The project focuses on enabling clients to query relational databases through a single POST route, with support for aggregation, joins, and strong typing via TypeScript inference.

The "BI" in the name stands for Business Intelligence, the discipline of analyzing data to support decision-making. The prefix "Flower" was selected by the project's original author and is not meant to be interpreted literally. For more details on the project's purpose and capabilities, see the [What is FlowerBI?](flowerbi-overview-what-is-flowerbi.md) documentation and the [main README](https://github.com/danielearwicker/flowerbi).

In summary, there are zero flowers in FlowerBI. It is a software tool, not a horticultural reference.
